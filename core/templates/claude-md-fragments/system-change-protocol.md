## System Change Protocol

**MANDATORY -- applies whenever you create, modify, or remove a skill, hook, spec, MCP server, or other system feature.**

STOP before claiming work is complete. You MUST:
1. Read `~/.claude/docs/system.md` and follow the System Change Checklist.
2. Every item on that checklist is non-negotiable. Do not skip items because they "probably don't apply."
3. If you created a new system feature -- something with behavior or workflow logic that a future Claude session would need to understand to modify correctly -- offer to create a spec.

Failure to follow this protocol means the work is NOT complete, even if the feature itself works.
